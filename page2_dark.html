Unit 2

Class Structure and Design

Overview

Unit 2 delves deeper into classes and objects, showing how you can create an object with its own set of variables, then change or access those variables elsewhere.

Constructors

There are two types of constructors. These are No-Argument Constructors and Paramaterized Constructors. No-Argument Constructors set the default variables when creating an object from a class. Paramaterized Constructors take input values when instantiating a class to create an object. These values are called instance variables. These instance variables are then transferred to local variables in the newly created object. You can also use constructors to change static variables, but that's something you'll learn about later. A constructor is created by creating a method in a class that is has the name of the class, such as "public className(instance variables)".

Mutators

Mutators, also known as Setters, allow you to change the variables stored in an object. They are written as a method with a return type of void, as you are trying to change a variable, not access one. They do, however, have an input value of the same type as the variable that you are trying to change. The method typically is named by using Set, followed by the variable name. For example, if you are trying to change a String called name, you would name the method setName.

Accessors

Accessors, also known as Getters, allow you to read the variables set by the Constructor or Mutator. They are the opposite of a Mutator in that they do not have an input value, but they do have a return type of whatever variable you are trying to access. This type of method follows a similar naming convention to Mutators, only with get instead of set. So going back to the String name, you would name the method getName.
